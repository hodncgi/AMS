## POST /nbq/next

Select the next best intake question based on client profile.

**Request:**
```json
{
  "profile": {
    "sector": "Healthcare",
    "solution": "Custom"
  }
}
```

**Response:**
```json
{
  "id": "q2",
  "text": "Where are HIPAA audit logs stored?",
  "fields": ["textarea"],
  "weight": 70,
  "why": "HIPAA compliance requires audit log visibility"
}
```

**Business rules (agreed Week 2):**
- Each question has a base weight in a fixed pool  
- Healthcare + security category ‚Üí +50 weight boost  
- `why` generated by Mistral with rule-based fallback

## POST /change-risk

Compute a **change risk score (0‚Äď100)** ‚ÄĒ decision-support only.

**Request:**
```json
{
  "criticality": "high",
  "test_days": 10
}
```

| Field | Type | Values |
|-------|------|--------|
| `criticality` | string | `low` (+20), `medium` (+50), `high` (+80) |
| `test_days` | integer | ‚Č• 0, each day reduces risk by 5 points |

**Response:**
```json
{
  "score": 30,
  "drivers": [
    "High criticality: +80 risk",
    "Test days (10): -50 risk"
  ]
}
```

Score clamped to minimum 0. Final acceptance remains with AMS leads.

## POST /chat

**Request:**
```json
{
  "messages": [
    { "role": "user", "content": "What is AMS intake?" }
  ]
}
```

**Response (planned):**
```json
{
  "reply": "...",
  "sources": ["AMS Intake Playbook"],
  "mode": "mistral",
  "retrieval_method": "keyword",
  "response_time_ms": 1200
}
```

**Modes:** `mistral` (full AI) | `fallback` (knowledge base only, LM Studio offline)
